# Denoising Diffusion Models — From Zero to DDPM

A clear, code-first journey from a naive image predictor to modern diffusion models.

---

## What is this?

This repository is a step-by-step educational series of Jupyter notebooks that build diffusion models from scratch.

Starting from the simplest possible idea ("let's add noise and then remove it"), we gradually discover why modern diffusion models work the way they do — without skipping the math or hiding it behind black-box libraries.

Each notebook introduces one new concept and builds runnable code. By the end, you'll have implemented a working DDPM (Denoising Diffusion Probabilistic Model) on CIFAR-10.

## How This Series Is Different

Most tutorials do one of two things:

- Too simple: "Look, we used diffusers and it works!" (you learn nothing)
- Too math-heavy: Equations without intuition (you give up)

This series takes a third path:

Every mathematical concept is introduced through code first, then explained with analogies and visualizations.

For example, instead of throwing alpha_bar_t = product of alpha_i at you, we build it step by step:

alpha_bar = torch.cumprod(alpha, dim=0)
x_t = sqrt(alpha_bar[t]) * x_0 + sqrt(1 - alpha_bar[t]) * noise

Then we explain why the square roots are there (scaling standard deviation, not variance). No magic. No "it's just a hyperparameter."

## Notebook Structure

Every notebook follows the same pattern:

1. Markdown cells (English) — intuitive explanations with metaphors, questions, and humor
2. Code cells — runnable Python that does exactly what the text describes
3. Visualizations — plots and images at every meaningful step
4. Summary — connecting what you just learned to the next chapter

Nothing is hidden. You see every line of code, every equation, every failed attempt (and why it failed).

## FAQ

Q: Do I need a GPU?
A: For training from scratch, yes (even a T4 on Colab works). For following along and understanding the code, CPU is fine.

Q: How long does each notebook take to run?
A: The early ones run in seconds. Chapter 6 (DDPM) takes about 30 minutes on a T4 for basic training. but if you have a GPU, use bigger model, bigger batch and smaller learning rate.

Q: I see a bug. What do I do?
A: Open an issue or submit a PR. This is a learning resource — corrections are welcome.
